#  BridgeLingo
### AI-Powered Kinyarwanda Language Learning App

> *Bridge the gap between languages — one lesson at a time.*

---

##  Overview

**BridgeLingo** is a full-stack mobile application built with **Flutter** and **Firebase** that helps users learn Kinyarwanda — the national language of Rwanda. Inspired by modern language-learning apps, BridgeLingo features structured courses, gamified challenge levels, an AI tutor chatbot, progress tracking, and downloadable certificates of completion.

This project was developed as a final mobile development project, demonstrating mastery of Flutter clean architecture, Firebase backend integration, advanced state management with Riverpod, and professional UI/UX design.

---

##  Features

| Feature | Description |
|---|---|
|  **Authentication** | Email/Password + Google Sign-In via Firebase Auth |
|  **Courses** | Structured lessons with flashcards and quizzes |
|  **Challenge Path** | 20 progressive levels that unlock on completion |
|  **AI Tutor** | Interactive Kinyarwanda chatbot for practice |
|  **Progress Tracking** | XP, streak, and lesson completion stored in Firestore |
|  **Certificates** | Downloadable PDF certificates on course completion |
|  **Theme Support** | Light, Dark, and System theme via SharedPreferences |
|  **Language Preferences** | Native language selection persisted locally |
|  **Notifications Toggle** | Daily reminder preference saved with SharedPreferences |
|  **Edit Profile** | Update display name synced to Firebase Auth + Firestore |

##  Architecture

BridgeLingo follows **Flutter Clean Architecture** with a clear separation of concerns:

```
lib/
├── core/
│   ├── components/        # Reusable widgets (GradientButton, CustomTextField)
│   ├── presentation/      # Shell scaffold with bottom navigation
│   ├── router/            # GoRouter with auth guard redirects
│   ├── services/          # DataSeeder — seeds Firestore on first launch
│   └── theme/             # AppTheme + ThemeModeNotifier (SharedPreferences)
│
├── features/
│   ├── auth/
│   │   ├── data/          # AuthRepository — Firebase Auth + Firestore
│   │   └── presentation/  # Login, Register, ForgotPassword, Welcome screens
│   │
│   ├── dashboard/
│   │   ├── data/          # CourseRepository (Firestore CRUD), UserStatsProvider
│   │   ├── domain/        # Course, Lesson, Exercise, Certificate models
│   │   └── presentation/  # Dashboard, Profile, EditProfile, Challenges, Chat
│   │
│   ├── lesson/            # LessonList + LessonPlayer (flashcards + quizzes)
│   └── settings/          # Language, theme, notifications preferences
│
├── firebase_options.dart  # Auto-generated by FlutterFire CLI
└── main.dart              # Entry point — Firebase init + DataSeeder
```

**State Management:** [Riverpod](https://riverpod.dev/) — all business logic lives in providers and repositories, never in UI widgets.

**Navigation:** [GoRouter](https://pub.dev/packages/go_router) with auth-aware redirect guards.

##  Firebase Architecture

### Collections

```
firestore/
├── users/
│   └── {uid}/
│       ├── fullName, email, nativeLanguage, photoUrl, createdAt
│       ├── progress/
│       │   └── {courseId}/  — progress (double), updatedAt
│       ├── certificates/
│       │   └── {id}/        — courseId, courseName, issuedAt
│       └── challenge_results/
│           └── {id}/        — level, success, completedAt
│
├── courses/
│   └── {courseId}/
│       ├── title, description, level, imageUrl, createdAt
│       └── lessons/
│           └── {lessonId}/
│               ├── title, content, orderIndex
│               └── exercises/
│                   └── {id}/ — type, question, options[], correctAnswer
│
└── challenges/
    └── {id}/  — level, type, question, options[], correctAnswer
```

### Authentication Methods
- Email / Password (with email verification)
- Google Sign-In

### Security Rules
```js
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Users can only read/write their own data
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;

      match /progress/{doc} {
        allow read, write: if request.auth != null && request.auth.uid == userId;
      }
      match /certificates/{doc} {
        allow read, write: if request.auth != null && request.auth.uid == userId;
      }
      match /challenge_results/{doc} {
        allow read, write: if request.auth != null && request.auth.uid == userId;
      }
    }

    // Courses are readable by all authenticated users
    match /courses/{courseId} {
      allow read: if request.auth != null;
      allow write: if false;

      match /lessons/{lessonId} {
        allow read: if request.auth != null;
        match /exercises/{exerciseId} {
          allow read: if request.auth != null;
        }
      }
    }

    // Challenges are readable by all authenticated users
    match /challenges/{challengeId} {
      allow read: if request.auth != null;
      allow write: if false;
    }
  }
}
```

##  Getting Started

### Prerequisites

| Tool | Version |
|---|---|
| Flutter SDK | 3.x or later |
| Dart SDK | 3.2.0 or later |
| Android Studio | Latest |
| Firebase CLI | Latest |
| Java | 17+ |

### 1. Clone the repository

```bash
git clone https://github.com/Noella-Ntare/bridgelingo.git
cd bridgelingo/frontend
```

### 2. Install dependencies

```bash
flutter pub get
```

### 3. Firebase Setup

> **You need your own Firebase project.** The `google-services.json` is not included in this repo for security reasons.

1. Go to [Firebase Console](https://console.firebase.google.com)
2. Create a project named `bridgelingo`
3. Enable **Authentication** → Email/Password + Google
4. Create a **Firestore Database** in `europe-west1`
5. Register an Android app with package name `com.example.bridgelingo`
6. Download `google-services.json` → place at `android/app/google-services.json`
7. Run FlutterFire CLI:

```bash
dart pub global activate flutterfire_cli
flutterfire configure
```

### 4. Apply Firestore Security Rules

Copy the security rules from the section above into your Firebase Console under **Firestore → Rules**.

### 5. Run the app

```bash
# On Android emulator or physical device
flutter run

# Release build
flutter run --release
```

>  **Important:** Web, Desktop, and Chrome builds will not work as intended. This is a **Mobile Application** — run on Android emulator or physical device only.

##  Known Limitations & Future Work

- **AI Tutor** currently uses a local rule-based response engine. Future versions will integrate a live LLM API (e.g., Gemini or OpenAI) for dynamic conversations.
- **Audio pronunciation** is planned but not yet implemented — exercises are text-based only.
- **Admin panel** for adding courses and challenges without the DataSeeder is planned.
- **iOS support** — the app is Android-only in this submission. iOS configuration requires additional setup.
- **Offline mode** — Firestore offline persistence is not yet explicitly configured; network access is currently required.
